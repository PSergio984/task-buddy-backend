# Plan: 01-02 CRUD Refactoring & Enum Standardization

**Goal**: Decouple CRUD from routers and remove hardcoded literals.

## Proposed Changes

### 1. Standardize Enums
- **[NEW]** `app/schemas/enums.py`
  - `TaskStatus`
  - `AuditAction`
  - `ResourceType`
- Update ORM models and Pydantic schemas to use these Enums.

### 2. Implement CRUD Layer
- **[NEW]** `app/crud/user.py`
- **[NEW]** `app/crud/task.py`
- **[NEW]** `app/crud/tag.py`
- Functions: `get`, `get_multi`, `create`, `update`, `delete`.
- **Constraint**: No `db.commit()` inside these functions.

### 3. Refactor Routers
- **[MODIFY]** `app/api/routers/task.py`
- **[MODIFY]** `app/api/routers/user.py`
- **[MODIFY]** `app/api/routers/audit.py`
- Inject `db: AsyncSession = Depends(get_db)`.
- Replace direct `tbl_*.select()` with CRUD calls.
- Add `await db.commit()` and `await db.refresh(obj)` in the router endpoints.

## Verification Plan

### Automated Tests
- `pytest` — Full suite must pass.
- Verify that multiple CRUD calls can be wrapped in a single `db.commit()`.

### Manual Verification
- Swagger UI (`/docs`) — Test all CRUD operations for Tasks, Subtasks, Tags, and Audit Logs.
- Verify Enums are correctly reflected in the API schema.
