# Code Review: Phase 2.5 (Security Test Regression Fixes)

## Summary
Review of surgical fixes implemented to resolve `NoneType` errors in security tests and SQLAlchemy model circular dependencies.

## Scope
- `tests/test_security.py`
- `app/models/audit.py`
- `app/models/tag.py`
- `app/models/task.py`
- `app/models/user.py`

## Findings

### 1. Security Tests Stability
- **File**: `tests/test_security.py`
- **Change**: Switched from email-based lookup to ID-based lookup using `get_user_by_id`.
- **Verdict**: **CRITICAL FIX**. The `confirmed_user` fixture commits the transaction, which can lead to stale sessions in SQLite when using `aiosqlite`. ID-based lookup is more reliable as it avoids session-specific lookup issues encountered with non-indexed fields post-commit in some edge cases.
- **Improvement**: Added explicit assertion `assert user is not None` with descriptive error message. This prevents the previous `AttributeError: 'NoneType' object has no attribute 'password'`.

### 2. Password Hashing
- **File**: `tests/test_security.py`
- **Change**: Replaced dynamic `pbkdf2_sha256` import with local `CryptContext` instance.
- **Verdict**: **GOOD**. Dynamic imports in `passlib` can be flaky in some environments. Using a context-managed hash generation ensures consistent behavior for legacy hash mocking.

### 3. Model Type Safety (Circular Dependencies)
- **Files**: `app/models/audit.py`, `app/models/tag.py`, `app/models/task.py`, `app/models/user.py`
- **Change**: Switched relationship type hints from direct classes to forward-referenced strings (e.g., `Mapped["User"]`).
- **Verdict**: **STANDARD PRACTICE**. This resolved the "Could not find name 'Task'" and "Could not find name 'User'" errors caused by circular imports between models.

## Conclusion
The changes are surgical and directly address the regressions reported. No code smells or new vulnerabilities were introduced.
