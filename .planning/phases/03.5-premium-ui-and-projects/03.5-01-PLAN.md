# Plan 03.5-01: Backend Project Refactor (Terminology Alignment)

## Objective
Refactor the backend to replace all instances of "Group" with "Project" and "group" with "project" to align terminology with the new frontend design.

## Context
The application is moving from "Groups" to "Projects" for better UX clarity. The backend needs to be updated to reflect this change in models, schemas, CRUD operations, and API endpoints.

## Tasks

### 1. Update Models
- [ ] Rename `app/models/group.py` to `app/models/project.py`.
- [ ] In `app/models/project.py`: Rename class `Group` -> `Project`, table `tbl_groups` -> `tbl_projects`.
- [ ] In `app/models/task.py`: Rename `group_id` -> `project_id`, relationship `group` -> `project`.
- [ ] In `app/models/user.py`: Rename relationship `groups` -> `projects`.
- [ ] Update `app/models/__init__.py` (if it exists) to reflect renames.

### 2. Update Schemas
- [ ] Rename `app/schemas/group.py` to `app/schemas/project.py`.
- [ ] In `app/schemas/project.py`: Rename all classes (e.g., `GroupBase` -> `ProjectBase`, `GroupCreate` -> `ProjectCreate`).
- [ ] Update `app/schemas/__init__.py` (if it exists).

### 3. Update CRUD Operations
- [ ] Rename `app/crud/group.py` to `app/crud/project.py`.
- [ ] In `app/crud/project.py`: Update function names and internal references (e.g., `get_group` -> `get_project`).

### 4. Update API Routers
- [ ] Rename `app/api/routers/group.py` to `app/api/routers/project.py`.
- [ ] In `app/api/routers/project.py`: Update router tags, prefix, and logic.
- [ ] Update `app/main.py`: Update router registration (`app.include_router`).

### 5. Update Seed Script and Misc
- [ ] Update `scripts/seed.py` (or `seed.py` in root) to use `Project` terminology.
- [ ] Update any other references in the codebase (e.g., `app/api/deps.py`, `app/core/config.py`).
- [ ] Rename variables like `db_group` -> `db_project`, `group_in` -> `project_in`, `GROUP_NOT_FOUND` -> `PROJECT_NOT_FOUND`.

### 6. Verification
- [ ] Run the seed script: `python seed.py`
- [ ] Run tests (if any) to ensure no regressions.

## Success Criteria
- Codebase uses "Project" instead of "Group".
- Database tables are renamed (if applicable, though we might rely on SQLAlchemy/Alembic or just fresh start).
- API endpoints are `/projects` instead of `/groups`.
- Seed script runs successfully.
