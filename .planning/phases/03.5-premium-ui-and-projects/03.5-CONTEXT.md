# Phase 3.5: Premium UI, Projects, and Data Seeding

## Domain
This phase focuses on frontend UI/UX improvements, specifically targeting the sidebar, the Create Task flow, and the overall aesthetic. It introduces the new "Projects" (Workspaces) functionality to organize tasks, and implements a backend Python script to populate realistic dummy data. 

## Decisions

### 1. Create Task Modal & Calendar
- **Design:** Remove the existing grey color scheme. Apply a high-end, clean aesthetic.
- **Calendar Library:** Use `react-day-picker` (following Shadcn UI patterns) for a robust and premium date selection experience.

### 2. Projects (Workspaces) Functionality
- **Implementation:** Users will have the ability to create "Projects".
- **Usage:** Tasks can be assigned to these projects (providing the requested list/grouping functionality).
- **Sidebar Constraints:** Keep the sidebar simple for now. Do NOT add label filters to the sidebar (those will be reserved for the task page).

### 3. Sidebar Fixes
- **Broken Icon:** Fix the collapse icon in the sidebar so it works correctly and smoothly.

### 4. Dummy Data / Seeding
- **Backend Script:** Create a Python seed script in the backend repository.
- **Data Generation:** The script will populate the database with realistic Projects (Workspaces), Tasks, and Subtasks to make the application feel populated and alive during development and testing.

### 5. Premium Design Implementation
- **Skill Application:** Strict adherence to the `premium-frontend-ui` skill.
- **Visuals & Motion:** Utilize modern design principles such as glassmorphism where appropriate, fluid typography, high-contrast layouts, and smooth animations (e.g., Framer Motion / Lenis for scroll smoothing if applicable). 
- **Performance:** Ensure animations and design elements do not compromise performance.

### 6. Measurable Premium UI Acceptance Criteria

These criteria make the "Review & Polish" step grep-verifiable (added per 03.5-REVIEWS.md):

**Animations:**
- Sidebar collapse: `duration ≤ 300ms`, easing `ease-out` or spring `stiffness ≥ 200`
- Task card hover: `translateY(-2px)` with `transition: 150ms ease`
- Modal open/close: fade + scale, `duration 200ms`

**Glassmorphism (where applied):**
- Background: `rgba(255,255,255,0.08)` to `rgba(255,255,255,0.12)`
- Blur: `backdrop-filter: blur(12px)` minimum
- Border: `1px solid rgba(255,255,255,0.15)`

**Typography:**
- Body: Inter or equivalent, `font-size: 14px`, `line-height: 1.6`
- Headings: `font-weight: 600+`, `letter-spacing: -0.02em` on sizes ≥ 24px

**Contrast:**
- All body text meets WCAG AA (≥ 4.5:1 ratio)
- Primary CTA button: contrast ≥ 4.5:1 against its background

**Date Picker:**
- Confirmed by `import { DayPicker }` or `from 'react-day-picker'` in source
- Selected date must be visible in the task form field after selection
