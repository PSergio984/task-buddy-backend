# Phase 4.9 Context: Technical Debt Cleanup

## Overview
This phase addresses critical technical debt identified during the transition between Phase 4 and Phase 5. The focus is on ensuring production reliability for database schema changes and fixing core usability bugs in the frontend filtering system.

## Decisions

### 1. Backend: Alembic Migration Restoration
- **Problem:** The rename from `Groups` to `Projects` was handled by a database clear.
- **Decision:** Create a "bridge" migration or a synthetic historical migration that correctly handles `op.rename_table` for `tbl_groups` -> `tbl_projects` and updates all foreign key constraints.
- **Goal:** A clean `alembic upgrade head` should work on a database that still has `tbl_groups`.

### 2. Frontend: Multi-Dimension Filtering
- **Problem:** Sidebar filters (Project/Category) and Dashboard filters (Status/Timeframe) currently conflict or overwrite each other.
- **Decision:**
    - Refactor `MainLayout` to maintain independent state for `sidebarFilter` (project ID or category name) and `statusFilter`.
    - Update `useTasks` hook (or equivalent) to accept both filters simultaneously.
    - Ensure the UI reflects active filters correctly (e.g., highlighting both the selected Project in the sidebar and the "Pending" tab in the dashboard).

### 3. Cleanup: Dead Code & Type Safety
- **Decision:** 
    - Remove unused imports identified in `CONCERNS.md` (e.g., `user`, `LogOut` in `sidebar.tsx`).
    - Fix `any` types in `new-task-modal.tsx` and other core components.
    - Add missing `aria-labels` for accessibility.

## Success Criteria
- [ ] Backend: A fresh DB can be initialized AND an old DB (with `tbl_groups`) can be upgraded to the current schema.
- [ ] Frontend: Users can filter by "Project: Work" AND "Status: Pending" simultaneously.
- [ ] Frontend: No `any` types remaining in core components (`TaskCard`, `Dashboard`, `NewTaskModal`).
- [ ] Frontend: Dashboard no longer renders an empty state when a valid sidebar filter is selected.

## Out of Scope
- Implementing WebSockets/SSE (Deferred to Phase 5).
- Major UI redesigns.
